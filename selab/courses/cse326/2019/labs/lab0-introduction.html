    What is VCS (Version Control System)?

    • Database for Version
    • A system for managing multiple revisions of the same unit of information
    • Git, SVN, Mercurial, ...
    • Why use VCS?
      • to collaborate with team
      • to backup the version of project
      • to track changes in source code
      • to reduce side-effects when developing a new module

    Git

    • the most commonly used VCS
    • Git Tutorial: https://rogerdudler.github.io/git-guide/index.ko.html (in Korean)
    • Download: https://git-scm.com/

    GitHub

    • Web based remote source code repository
    • offers git repository hosting service
    • https://github.com/

    Git: Create GitHub Remote Repository

    • If you are new GitHub user, click Start a project.

    • If you are already a registered GitHub user, click New repository.

    Git: Create GitHub Remote Repository

    Git: Create GitHub Remote Repository

    • Copy your repository URL for next step.

    Git: Initial Setting

    1. Initialize Git Workspace: git init
    2. User login:
      • git config user.name "Your Name"
      • git config user.email yourGitEmail@abcde.def (No double quotes!)
    3. Naming your GitHub Remote Repository: git remote add origin https://github.com/...

    Git: Basic Workflow

    1. Pull your remote repository to Local Repository and Workspace
    2. Code, Code, Code ... in your Workspace
    3. Add your changes to Index
    4. Commit(Save) your changes to Local Repository
    5. Pull your remote repository again
      • conflict can occur when remote repository is modified before push
    6. Upload your works in Local Repository to Remote Repository

    Git: Basic Workflow (Commands)

    1. git pull origin master
    2. Code, Code, Code ... in your Workspace
    3. git add *
    4. git commit -m "Describe how code is changed ..."
    5. git pull origin master
    6. git push origin master

    Environment Setup: MAMP

    • In order to practice the concepts and theories learned in this course, web server must be setup.
    • Before the course web server gets configured, we will use MAMP.

    • MAMP (Mac OS X + Apache + MySQL + PHP) consists of :
      • Apache HTTP/HTTPS Server - Web server application
      • MySQL - Open-source Relation Database Management System (RDBMS)
      • PHP - Server-side scripting Language
    • It setups local server environment and supports both Mac and Windows.
    • Download: https://www.mamp.info/en/downloads/

    Ex 1: Setup Git & Create GitHub Repository

    1. Download Git and install it.
      • Don't change any settings when you install Git.
    2. Create new GitHub Repository.
    3. Open Terminal.
      • Windows: Win+R -> cmd
      • Mac OS X: open Spotlight -> search 'Terminal' ('터미널')
    4. Follow the Git Basic Workflow(slide 9-10).
      • Initialize ~ Pull -> Create some file(anything is okay) -> Add ~ Push
      • Type the commands in Terminal or Git Bash.

    Ex 2: Setup MAMP (Web Server & MySQL)

    1. Download MAMP and install it.
      • Don't install MAMP PRO.
    2. Execute MAMP and click Start Servers.
      • If there is an error, restart MAMP, start server, stop server(to terminate the service), and start server again.
    3. Open your Internet browser (IE, Chrome, Firefox, ...).
    4. Type localhost or 127.0.0.1 in address.
    5. Type mysql -uroot -proot in Terminal.

    Ex 3: Creating a demo web page

    • Atom
      • it is advisable to use Atom as an editor for web application development
    • Recreate the Hello World demo page using Atom
      1. browse the Hello World demo page
      2. view the page source
      3. recreate the demo page using Atom
      4. open and check your web page in a web browser
    • Upload your web page
      • put Hello World demo page in the the Apache root directory
      • browse the uploaded demo page at http://localhost/helloWorld.html
      • download the server-side script and put it in the the Apache root directory
      • browse the uploaded script at http://localhost/welcomeToCSE3026.php?name=yourname
        • e.g.) http://localhost/welcomeToCSE3026.php?name=Scott
